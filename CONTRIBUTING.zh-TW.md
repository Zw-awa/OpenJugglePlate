# 參與 OpenJugglePlate

[English](./CONTRIBUTING.md) | [简体中文](./CONTRIBUTING.zh-CN.md) | [繁體中文](./CONTRIBUTING.zh-TW.md)

感謝你考慮為 OpenJugglePlate 做出貢獻。
這是一個開源硬體機器人專案，因此有價值的貢獻不只限於程式碼。

我們歡迎以下方向的改進：

- 機械與 CAD
- 韌體
- Linux SBC 側軟體
- 控制設計
- 標定與測試流程
- BOM 與採購說明
- 文件與重現性建設

## 開始之前

如果改動較大，請先開一個 issue。
這樣可以避免改動偏離目前的硬體假設、路線安排或安全包絡。

建議先閱讀：

- [`README.md`](./README.md)

## 貢獻原則

你的貢獻最好至少改善以下其中一個面向：

- 系統清晰度
- 可重現性
- 安全性
- 可維護性
- 有證據支撐的性能提升

請盡量避免「只增加複雜度，但沒有讓建構、理解、驗證或擴展更容易」的改動。

## Pull Request 的基本要求

### 文件改動

如果主要改的是文件，請說明：

- 原本哪裡不清楚或缺失
- 具體改了什麼
- 是否同步更新了截圖、示意圖或連結

### 硬體改動

如果涉及機械、電氣或 BOM，請說明：

- 設計意圖
- 影響了哪個子系統
- 哪些尺寸、介面或假設發生了變化
- 是否還需要同步更新裝配說明或採購說明

### 韌體或軟體改動

如果涉及程式碼，請說明：

- 改變了什麼行為
- 你如何驗證它
- 你依賴了哪些硬體或環境假設
- 是否還應同步更新日誌、文件或測試

## 安全與範圍

這個專案涉及會運動的硬體。
如果你的改動影響了運動限幅、電源、歸零、致動器或安全行為，請在 PR 中明確指出。

不要悄悄改動以下內容：

- 運行假設
- 目標物件
- 與安全相關的預設值
- `Linux SBC` 與 `STM32` 之間的職責邊界

## 風格要求

- 優先清晰、直接的表達，不要寫成宣傳文案
- 公開文件應盡量讓只擅長某一個方向的貢獻者也能看懂
- 優先做漸進式改動，而不是大而空的重寫
- 如果某個決策從 diff 上看不明顯，請補充原因

## 提問與支援

如果你不確定從哪裡開始，可以開一個 support/question issue，並說明：

- 你想參與哪一部分
- 你目前已掌握哪些上下文
- 現在卡住你的問題是什麼
