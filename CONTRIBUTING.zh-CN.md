# 参与 OpenJugglePlate

[English](./CONTRIBUTING.md) | [简体中文](./CONTRIBUTING.zh-CN.md) | [繁體中文](./CONTRIBUTING.zh-TW.md)

感谢你考虑为 OpenJugglePlate 做贡献。
这是一个开源硬件机器人项目，所以有价值的贡献不只限于代码。

我们欢迎以下方向的改进：

- 机械与 CAD
- 固件
- Linux SBC 侧软件
- 控制设计
- 标定与测试流程
- BOM 与采购说明
- 文档与复现性建设

## 开始之前

如果改动较大，请先开一个 issue。
这样可以避免改动偏离当前的硬件假设、路线安排或安全包络。

推荐先看的内容：

- [`README.md`](./README.md)

## 贡献原则

你的贡献最好至少改善以下一个维度：

- 系统清晰度
- 可复现性
- 安全性
- 可维护性
- 有证据支持的性能提升

请尽量避免“只增加复杂度，但没有让构建、理解、验证或扩展更容易”的改动。

## Pull Request 的基本要求

### 文档改动

如果主要改的是文档，请说明：

- 原来哪里不清楚或缺失
- 具体改了什么
- 是否同步更新了截图、示意图或链接

### 硬件改动

如果涉及机械、电气或 BOM，请说明：

- 设计意图
- 影响了哪个子系统
- 哪些尺寸、接口或假设发生了变化
- 是否还需要同步更新装配说明或采购说明

### 固件或软件改动

如果涉及代码，请说明：

- 改变了什么行为
- 你如何验证它
- 你依赖了哪些硬件或环境假设
- 是否还应该同步更新日志、文档或测试

## 安全与范围

这个项目涉及会运动的硬件。
如果你的改动影响了运动限幅、电源、归零、执行器或安全行为，请在 PR 中明确指出。

不要悄悄改动以下内容：

- 运行假设
- 目标对象
- 与安全相关的默认值
- `Linux SBC` 与 `STM32` 之间的职责边界

## 风格要求

- 优先清晰、直接的表达，不要写成宣传文案
- 公开文档应尽量让只擅长某一个方向的贡献者也能读懂
- 优先做渐进式改动，而不是大而空的重写
- 如果一个决策从 diff 上看不明显，请补充原因

## 提问与支持

如果你不确定从哪里开始，可以开一个 support/question issue，并说明：

- 你想参与哪一部分
- 你目前已经掌握了哪些上下文
- 当前卡住你的问题是什么
